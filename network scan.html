Potential Security Risks

Port Service Risk Why / Short Mitigation
21/tcpftpHighFTP transmits credentials in cleartext. Mitigation: Disable FTP or replace with SFTP/FTPS; restrict access by IP; strong creds.
22/tcpsshMediumSSH is common remote admin vector — safe if hardened. Mitigation: Use key-based auth, disable password logins, change port or use port-knocking, fail2ban, restrict source IPs.
23/tcptelnetCriticalTelnet is unencrypted and easily abused. Mitigation: Disable Telnet; use SSH instead; block on firewall.
25/tcpsmtpMediumMail servers can be abused for spam or exploited. Mitigation: Require auth/relay restrictions, patch MTA, monitor logs, restrict port to known mail gateways.
53/tcpdomain (DNS)HighOpen DNS can be used for cache poisoning, DNS amplification. Mitigation: Run authoritative/recursive separation; disable open recursion; rate-limit and restrict queries.
80/tcphttpMediumHTTP may expose web apps with vulnerabilities. Mitigation: Use HTTPS, patch web servers, WAF, scan web apps, remove dev/test pages.
111/tcprpcbindHighrpcbind/portmapper can expose RPC services and lead to remote exploits. Mitigation: Disable if unused; firewall RPC ports; restrict to trusted hosts.
139/tcpnetbios-ssnHighSMB/NetBIOS exposes file shares and credentials. Mitigation: Block on WAN, disable NetBIOS over TCP if not needed, enforce SMB signing, patch.
445/tcpmicrosoft-ds (SMB)CriticalSMB is high-risk (ransomware/remote code exec history). Mitigation: Block from untrusted networks, apply MS patches, limit shares, use strong auth and monitoring.
512/tcpexec (rlogin/rsh)Criticalrsh/rlogins are insecure and allow remote exec without encryption. Mitigation: Disable rsh/rlogin; use SSH.
513/tcplogin (rlogin)CriticalSame as above — insecure legacy remote login. Mitigation: Disable; use SSH.
514/tcpshell (rsh/syslog)HighLegacy remote shell services and syslog implications. Mitigation: Disable insecure services; use secure logging and SSH.
1099/tcprmiregistry (RMI)HighJava RMI can expose remote objects; often misconfigured and vulnerable. Mitigation: Disable or tunnel over SSH/VPN, enforce strong passwords and network restrictions.
1524/tcpingreslock (oracle/legacy)MediumLegacy DB or lock services may be unpatched. Mitigation: Remove legacy services, patch, firewall restrict.
2049/tcpnfsHighNFS can expose file systems; by default little auth for v3. Mitigation: Export only to trusted networks, use NFSv4 with auth (sec), firewall.
2121/tcpccproxy-ftp (alternate FTP)HighAlternate FTP services same FTP risks. Mitigation: Disable or secure (SFTP/FTPS), restrict access.
3306/tcpmysqlHighDatabases exposed to network are attractive target. Mitigation: Bind to localhost, use strong DB creds, restrict IPs, use TLS, audit/log.
5432/tcppostgresqlHighSame DB concerns as MySQL. Mitigation: Bind to localhost, firewall, strong auth, TLS, role hardening.
5900/tcpvncHighVNC often lacks strong auth and is unencrypted. Mitigation: Disable or tunnel over SSH/VPN, enforce strong passwords and network restrictions.
6000/tcpX11HighX11 remote display can allow input sniffing and code exec. Mitigation: Block from network, use SSH X11 forwarding instead.
6667/tcpircMediumIRC servers/clients can be abused or used as C2. Mitigation: Restrict access and monitor traffic.
8009/tcpajp13HighApache JServ Protocol (AJP) has had severe vulnerabilities (e.g., Ghostcat). Mitigation: Disable AJP if unused; bind to localhost; apply patches; restrict access.
8180/tcpunknown / http-altMediumLikely alternate HTTP/tomcat admin UI — can expose admin panels. Mitigation: Verify service, restrict access, enable auth, patch apps.

Port Service Risk Why / Short Mitigation
80/tcphttpMediumHTTP serves content unencrypted — attackers can intercept or tamper with traffic and web apps can expose vulnerabilities. Mitigation: Serve traffic over HTTPS (redirect 80 → 443), patch web server and web apps, remove default pages, use a WAF, harden and monitor logs.
7443/tcporacleas-https (alt HTTPS)Mediumfiltered indicates packets are blocked/filtered by a firewall/IPS (not responding). Port 7443 is often used for application-server HTTPS (Oracle/Tomcat). If reachable, it would expose management interfaces or app endpoints. Mitigation: Verify the service (identify banner/version), keep TLS configs and app server patched, restrict access to trusted IPs or VPN, confirm firewall rules intentionally filter/unfilter as required, and monitor attempts.
